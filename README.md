# linear-regression

Цель проекта - показать базовую практику линейной регрессии в `scikit-learn`: обучение модели, предсказание, `train/test split`, оценку качества через `MSE`, а также бонусную реализацию подбора коэффициентов без `scikit-learn` для основной логики.

## Что сделано

- Реализованы упражнения `ex00`-`ex05`.
- Основной формат решения - Jupyter notebooks.
- В каждом упражнении сначала идет основная часть под проверку, затем визуализация и краткие инженерные пояснения.
- В `ex04` дополнительно реализованы:
  - визуальная проверка качества модели через график `y_true vs y_pred` (выбрано потому, что такой график позволяет быстро увидеть, насколько предсказания близки к идеальной линии `y = x`: чем ближе точки к этой диагонали, тем точнее модель)
  - residual analysis, чтобы показать ошибки модели не только через `MSE`, но и через распределение остатков

- В `ex05` дополнительно по отношению к обязательной части проекта реализован бонусный блок:
  - brute-force поиск коэффициентов - полный перебор значений `a` и `b` на сетке с вычислением `MSE`, чтобы наглядно показать поверхность ошибки и точку минимума
  - gradient descent - итеративный поиск коэффициентов `a` и `b` через шаги по антиградиенту, чтобы показать, как модель приходит к минимуму без полного перебора
  - closed-form OLS - аналитическое решение линейной регрессии методом наименьших квадратов по формуле для коэффициентов `a` и `b`, добавленное как независимая проверка корректности (выбрано потому, что в задаче используется простая линейная регрессия `y = ax + b`, для которой можно получить точное решение напрямую и сверить с численными методами)
  - сравнение результата со `sklearn.LinearRegression`

## Запуск

### 1. Создать виртуальное окружение

```bash
python -m venv .venv
```

### 2. Активировать окружение

Windows:

```bash
source .venv/Scripts/activate
```

Linux / macOS:

```bash
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
python -m pip install -r requirements.txt
```

### 4. Выполнить все notebooks подряд

```bash
bash ./run_all.sh
```

### 5. Открыть notebooks вручную

```bash
python -m jupyter lab
```

## Кратко по упражнениям

### ex00 - Environment and libraries

Подготовка окружения и библиотек для проекта.

### ex01 - Scikit-learn estimator

Обучение `LinearRegression` на маленьком наборе данных, получение коэффициента, intercept, score и прогноза для нового значения.

### ex02 - Linear regression in 1D

Генерация одномерных данных через `make_regression`, построение scatter plot, обучение модели, получение уравнения прямой и ручной расчет `MSE`. Дополнительно показано влияние большего шума на качество регрессии.

### ex03 - Train test split

Разделение данных на train/test и визуализация разбиения.

### ex04 - Forecast diabetes progression

Прогнозирование target по diabetes dataset из `scikit-learn` через `load_diabetes(as_frame=True)`, обучение модели на всех признаках, анализ коэффициентов, предсказаний и `MSE` на train/test.

### ex05 - Gradient Descent (Optional)

Подбор коэффициентов линейной модели без `scikit-learn` в основной логике. Реализованы brute-force, gradient descent, closed-form OLS и сравнение со `LinearRegression` из `scikit-learn`.

## Особенности реализации

- Решение ориентировано на воспроизводимость и удобную ручную проверку.
- Визуализации добавлены не как декор, а как часть объяснения результата.
- Для `ex04` используется путь, который прямо указан в subject: `load_diabetes(as_frame=True)`.
- `run_all.sh` позволяет прогнать все notebooks сверху вниз одной командой.
